## Testing Commands

### Create Individual Categories:

```bash
# Create Italian Cuisine
curl -X POST http://localhost:5000/api/v1/categories \
  -H "Authorization: Bearer CHEF_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Italian Cuisine",
    "image": "https://images.unsplash.com/photo-1551183053-bf91a1d81141?w=500&h=300&fit=crop",
    "description": "Authentic Italian dishes featuring fresh pasta, wood-fired pizzas, and traditional Mediterranean flavors."
  }'
```

### Test Different Scenarios:

```bash
# Test duplicate name (should fail)
curl -X POST http://localhost:5000/api/v1/categories \
  -H "Authorization: Bearer CHEF_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Italian Cuisine",
    "description": "This should fail due to duplicate name"
  }'

# Test without authentication (should fail)
curl -X POST http://localhost:5000/api/v1/categories \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Test Category",
    "description": "This should fail due to no authentication"
  }'

# Test with customer role (should fail)
curl -X POST http://localhost:5000/api/v1/categories \
  -H "Authorization: Bearer CUSTOMER_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Test Category",
    "description": "This should fail due to insufficient permissions"
  }'
```

## Expected Response Format

When you create a category, you should receive a response like this:

```json
{
  "message": "Menu category created successfully",
  "menuCategory": {
    "_id": "65a1b2c3d4e5f6789012345",
    "name": "Italian Cuisine",
    "image": "https://images.unsplash.com/photo-1551183053-bf91a1d81141?w=500&h=300&fit=crop",
    "description": "Authentic Italian dishes featuring fresh pasta, wood-fired pizzas, and traditional Mediterranean flavors.",
    "menuCount": 0,
    "createdAt": "2024-01-20T10:30:00.000Z",
    "updatedAt": "2024-01-20T10:30:00.000Z"
  }
}
```

## Notes

- All images are from Unsplash and are free to use
- The `menuCount` field will be automatically managed by the system
- `createdAt` and `updatedAt` timestamps are automatically generated
- Category names must be unique across the system
- Only users with "chef" role can create, update, or delete categories
